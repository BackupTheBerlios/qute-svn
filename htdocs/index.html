Qute Utility for Tracking Errors

The goal of Qute is to provide APIs and tools to allow user-customized interaction with legacy web services, for example a command-line tool to access Bugzilla or a Vim interface to a Wiki.

The most interesting thing we've got so far is our source repository.

Current Status

Qute is currently made up of two Ruby libraries. qute.rb provides an API for interacting with web services, making it easy to write programs that fill out and submit forms, collect information from HTML tables, and follow hyperlinks. qutecmd.rb adds features useful for command-line tools, such as formatting ASCII result tables and filling out web forms at a command prompt.

These modules have proven themselves by providing the foundation for front-ends to two very different bug tracking web applications. Unfortuantely, both Qute-based tools are only used internally for proprietary web apps by companies that have chosen not to release all of the code.

We are currently working to release as much relevant code as possible, after which we plan to begin working on an open-source front end for bugzilla.
