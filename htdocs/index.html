Qute Utility for Tracking Errors

The goal of Qute is to provide APIs and tools to allow user-customized interaction with legacy web services, for example a command-line tool to access Bugzilla or a Vim interface to a Wiki.

The most interesting thing we've got so far is our source repository.

Current Status

Qute is currently made up of two Ruby libraries. qute.rb provides an API for interacting with web services, making it easy to write programs that fill out and submit forms, collect information from HTML tables, and follow hyperlinks. qutecmd.rb adds features useful for command-line tools, such as formatting ASCII result tables and filling out web forms at a command prompt. Qute was developed in-house at HP, which has generously released it under the BSD license.

Qute has proven to be useful and robust as the foundation for tools that interact with two very different bug tracking web applications. However, these are proprietary, would not be generally useful outside the companies where they were developed, and will not be released. We are planning to use the Qute modules to develop front-end tools for bugzilla.

As of late September 2004, we're working to release and document the latest versions of the Qute modules.
